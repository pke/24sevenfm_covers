Code signing policy

Updated 21 July 2026

How the downloads on this site are built — and how anyone can check that a file really is what it claims to be.

The downloads are not signed

Windows SmartScreen will warn the first time an installer runs. That warning is expected, and it is worth being straightforward about why rather than leaving you to guess.

A certificate from the SignPath Foundation, which provides free signing for open-source projects, was applied for in July 2026 and declined: the programme is for projects that have already built public visibility — stars, forks, contributors, external write-ups — and this one is new and small. That is a fair call, and the application can be renewed later. A commercial certificate costs several hundred euros a year, which is hard to justify for a free fan project.

So instead of asking you to trust a signature, everything below is arranged so you can verify the download yourself.

Verifying a download

Every release publishes a .sha256 sidecar next to each file. Compare it against what you downloaded:

sha256sum -c <file>.sha256 — or on Windows, certutil -hashfile <file> SHA256 and check it matches the sidecar.

That tells you the file wasn't altered in transit. For where it came from in the first place, every artifact is built by a public GitHub Actions run from the source in this repository — the run log names the exact commit, so the chain from source to download is inspectable end to end.

Team roles

This is a single-maintainer project, so one person holds every role. Naming that plainly is more honest than implying a review board that does not exist:

  • Author Philipp Kursawe (@pke) — writes the code and triggers the release build.
  • Reviewer Philipp Kursawe (@pke) — reviews what goes into a release.
  • Approver Philipp Kursawe (@pke) — decides what is released, and when.

Multi-factor authentication is enabled on the source repository, so the only account that can publish a release is protected by more than a password.

How a release is built

Every artifact is built from the source in this repository by a public GitHub Actions workflow — never on a developer's machine, and never uploaded by hand:

  • Source github.com/pke/24sevenfm_covers, MIT licensed. Every dependency is vendored into the repository as source, so a build pulls in nothing that isn't visible there.
  • Build The Release workflow runs the unit-test gate, compiles all three binaries, and publishes them as a GitHub release together with a .sha256 sidecar for every file.
  • Verify Compare a download against its .sha256 sidecar on the release page, and the workflow log shows exactly which commit produced it.

Privacy

The plugins and the standalone player collect no data, and this site sets no cookies and runs no analytics. The details, including every file written to your own machine, are in the privacy policy.
